# RoleFit High-Risk Review Harness

이 문서는 `my_harness`를 바로 설치하거나 자동 실행하지 않고, RoleFit Workbench에서 고위험 작업을 시작하기 전에 사람이 검토할 수 있는 체크리스트로 제한 적용하기 위한 템플릿입니다.

## When To Use

아래 작업에는 이 문서를 먼저 사용합니다.

- 실제 이력서, 후보자 정보, 회사 내부 문서가 들어갈 수 있는 작업
- PDF, Word, Excel, OCR, Document AI, 외부 provider 검토
- 매칭 점수 산식, 루브릭, 근거 추출 로직 변경
- 회사 API, Knox API, 인증, 메일 발송 연동
- GitHub 인수인계, Claude Code 인수인계, 배포 전 점검

아래 작업에는 사용하지 않습니다.

- 단순 문구 수정
- 작은 CSS 조정
- 테스트 문구 보정
- 단일 파일의 명확한 타입 오류 수정

## Fixed Rules

- `my_harness`를 현재 repo 루트에서 바로 실행하지 않습니다.
- `AGENTS.md`와 `CLAUDE.md`를 자동 생성물로 덮어쓰지 않습니다.
- `.claude`, `.codex`, `.agents` 경로에 생성물을 바로 배치하지 않습니다.
- 전역 `~/.codex/skills` 설치나 symlink를 만들지 않습니다.
- 외부 reviewer, 외부 LLM, 외부 OCR/API는 회사 승인 전 사용하지 않습니다.
- 실제 이력서, 후보자 개인정보, 회사 내부 문서, API key, Knox 인증정보를 입력으로 사용하지 않습니다.
- 자동 commit, push, PR 생성 권한을 하네스에 주지 않습니다.

## Data Safety Gate

작업 시작 전에 데이터 등급을 하나로 표시합니다.

```text
데이터 등급:
- [ ] 샘플/공개 데이터
- [ ] 내부 일반 문서
- [ ] 개인정보 포함 가능
- [ ] 비밀/API key/인증정보 포함 가능
```

기본 허용은 `샘플/공개 데이터`뿐입니다.

아래 항목이 있으면 하네스 사용을 중단하고 별도 보안 검토로 넘깁니다.

- 후보자 이름, 연락처, 이메일, 학력, 경력 원문
- 실제 이력서 또는 면접 기록
- 회사 내부 API URL, Knox 정보, 인증 방식 상세
- API key, token, password, credential
- 회사 내부 전략자료 원문

## Review Template

### 1. Goal

```text
이번 작업의 목표:

사용자가 성공했다고 판단할 기준:

이번 작업에서 하지 않을 것:
```

### 2. Current Context

```text
관련 파일 또는 문서:

현재 동작:

이미 검증된 것:

아직 모르는 것:
```

### 3. Risk Review

```text
요구사항/범위 리스크:

구현/테스트 리스크:

사용자 통제/보안 리스크:

개인정보 또는 회사 정보 노출 가능성:

되돌리기 어려운 작업 여부:
```

### 4. Execution Boundary

```text
수정 허용 파일:

수정 금지 파일:

외부 네트워크/API 사용 여부:

새 패키지 설치 여부:

커밋/푸시 필요 여부:
```

기본값은 다음과 같습니다.

- 외부 네트워크/API 사용 안 함
- 새 패키지 설치 안 함
- 커밋/푸시는 사용자 별도 요청 전까지 안 함
- 실제 개인정보와 회사 문서는 사용 안 함

### 5. Verification Plan

```text
좁은 테스트:

전체 테스트:

타입체크:

빌드:

수동 확인:

실패하면 되돌릴 기준:
```

RoleFit 기본 검증 명령은 다음입니다.

```powershell
npm.cmd test
npm.cmd run typecheck
npm.cmd run build
```

## Deep Review Output Format

고위험 작업 검토 결과는 아래 형식으로 남깁니다.

```md
## 판단
- 실행 가능 / 조건부 실행 가능 / 보류

## 주요 리스크
- ...

## 보완 권장사항
- ...

## 실행 경계
- 수정 허용:
- 수정 금지:
- 외부 호출:
- 사용자 승인 필요:

## 검증 체크리스트
- ...

## 롤백 기준
- ...

## 비개발자 설명
- 이번 작업은 쉽게 말해 ...
```

## my_harness Evaluation Rule

`my_harness` 산출물을 검토할 때는 다음 기준을 통과해야 합니다.

- 기존 `AGENTS.md`, `CLAUDE.md`와 충돌하지 않는다.
- 실행 권한을 넓히지 않는다.
- 외부 provider 또는 외부 reviewer를 기본값으로 켜지 않는다.
- 실제 후보자/회사 데이터를 요구하지 않는다.
- 생성물은 계획서, 체크리스트, 인수인계 문서 수준으로 제한된다.
- 스크립트, 설정 파일, 에이전트 지침, 의존성 파일은 기본 반영하지 않는다.

## 초딩 버전 설명

`my_harness`는 AI 팀을 만들어주는 도구입니다.

하지만 지금 바로 AI 팀을 전부 들이면 누가 뭘 하는지 헷갈릴 수 있습니다.

그래서 RoleFit에서는 먼저 이렇게만 씁니다.

```text
1. 위험한 작업인지 확인한다.
2. 개인정보나 회사 비밀이 있는지 확인한다.
3. 무엇을 고치고, 무엇은 안 고칠지 적는다.
4. 테스트 방법과 되돌리는 방법을 적는다.
5. 사람이 보고 승인한 뒤에만 구현한다.
```

핵심은 `my_harness`를 바로 실행하는 것이 아니라, 안전하게 일하기 위한 작업 점검표로만 쓰는 것입니다.
