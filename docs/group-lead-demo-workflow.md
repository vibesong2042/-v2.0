# 그룹장 보고용 데모 워크플로우

## 목적

RoleFit Workbench는 사내 메일 API와 임직원 검색 API를 연결하기 전에도, 채용 리포트가 현업부서 검토로 이어지는 흐름을 화면에서 설명할 수 있도록 데모 모드를 제공한다.

## 현재 데모에서 가능한 것

- 분석결과 리포트 생성 후 `현업부서 검토 요청` 패널 확인
- mock 부서장 주소록 검색 및 선택
- 부서장에게 보낼 검토 요청 메일 본문 미리보기
- 실제 메일 발송 없이 `Mock 검토 요청 발송` 상태 전환
- `부서장 화면 미리보기`에서 리포트 요약과 인터뷰 질문 확인
- 전화인터뷰 결과표 작성
- 실제 메일 발송 없이 `Mock 결과 회신` 상태 전환

## 아직 실제 연동이 아닌 것

- Knox 메일 API 실제 발송
- 회사 임직원 검색 API
- 부서장 인증 또는 권한 확인
- 전화인터뷰 결과 DB 저장
- 외부 링크 기반 부서장 전용 URL

## 보고 시 설명 문구

이 화면은 사내 API 연결 전 데모 모드입니다. 현재는 실제 메일을 보내지 않고, HR 담당자가 리포트를 만든 뒤 현업부서장에게 검토를 요청하고, 부서장이 전화인터뷰 결과를 다시 회신하는 업무 흐름을 검증하기 위한 화면입니다.

## 회사 API 연결 전 확인 항목

- 부서장 검색 API의 검색 조건과 응답 필드
- Knox 메일 API 인증 방식
- 테스트 발송 가능한 계정과 수신자
- 전화인터뷰 결과 저장 위치
- 개인정보/이력서 원문 저장 가능 여부
- 로그에 남기면 안 되는 항목

## 다음 구현 후보

- mock 부서장 목록을 회사 임직원 검색 adapter로 교체
- mock 메일 발송을 Knox 메일 adapter로 교체
- 부서장 검토 화면을 실제 링크 또는 인증된 라우트로 분리
- 전화인터뷰 결과표를 회사 표준 양식에 맞춰 저장/전송
